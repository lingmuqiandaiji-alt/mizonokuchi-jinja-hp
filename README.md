# 溝口神社公式ホームページ再構築計画

## 基本思想（AMAGI-CORE-001『デジタル継承基盤』）

本プロジェクトは「新しい溝口神社を作ること」ではなく、先人が築き、宮司・神職・氏子・
参拝者によって守られている『今の溝口神社』を尊重し、未来へ正しく受け継ぐための
デジタル基盤を整えることを目的とする。

- 判断に迷った場合の基準は常に「未来へ受け継げるか」
- デザインは神社の魅力を邪魔しない「器」であり、神社より目立ってはならない
- 写真は装飾ではなく「溝口神社の現在を未来へ残す記録」。派手さより神社らしさ・空気感・自然さを優先
- 「早く完成させる」より「長く使える」（5年後・10年後・20年後も更新し続けられる構造）
- ホームページの目的：参拝者の安心／神社の空気を伝える／迷わず情報へ辿り着ける／
  歴史を正しく伝える／次の世代が更新しやすい
- 制作順序：①基盤 → ②写真資産 → ③デザイン統一 → ④トップページ完成（通常＋季節・行事テーマ）→
  ⑤各ページ制作 → ⑥公開前品質確認
- 最終判断は千代吉本陣。判断が必要な事項は確認待ち一覧（CONTENT_INVENTORY.md）へ整理する

- 最終裁可者：千代吉本陣
- ホームページ管理者：天城さん
- 実装担当：Fable5
- 対象既存サイト：https://www.mizonokuchijinjya.org/

## 第1回納品物（本リポジトリ）

2026年水準の公式神社ホームページの初期基盤。トップページ（index.html）と、
今後の全ページ展開に耐えるCSS設計・運用文書一式。

## 動作確認方法

ブラウザで index.html を直接開くだけで表示できます（ビルド不要・外部依存なし）。
ローカルサーバーを使う場合：

```
python3 -m http.server 8000
# → http://localhost:8000
```

## 絶対方針（要点）

- 既存サイトの情報・機能・URLを削減しない（削除ではなく整理）
- Juyo Baseとは完全に別プロジェクト（コード・DB・認証・リポジトリを共有しない）
- 不明な情報は推測で確定せず、docs/CONTENT_INVENTORY.md の管理者確認項目として管理
- 本番公開・重大な方向転換は千代吉本陣の裁可が必要

## 技術方針

HTML5 / CSS3 / Vanilla JavaScript のみ。フレームワーク・外部CDN・Webフォント・有料サービス不使用。
JavaScript無効でも主要情報はすべて閲覧可能。
